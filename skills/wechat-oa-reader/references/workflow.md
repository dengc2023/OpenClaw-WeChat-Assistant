# 微信公众号（桌面端）自动化阅读：流程总结（模板匹配 + OCR + 坐标校准 + 滚动阅读）

本文记录从「微信主页面」到“读完并总结第一篇公众号文章”的**可复现**步骤、采用的方法与踩坑点。

## 0. 背景与目标
目标：在 macOS 微信桌面端中，完成以下闭环：
1) 搜索进入指定公众号
2) 进入公众号主页/历史文章列表（更全）
3) 在列表中定位目标文章并进入
4) 滚动读完全文并输出结构化总结

---

## 1) 从微信主页面找到公众号
### 步骤
- 激活微信 → `Cmd+F` 打开搜索
- 输入公众号关键词（如：`机器之心`）
- 用方向键在搜索结果里移动高亮（**不要立刻回车**）
- 通过“截图校验高亮项”确认当前高亮确实是目标公众号
- 回车进入公众号

### 方法
- **按键 → 截图 → 复核高亮 → 修正** 的视觉闭环。

### 坑点
- 纯靠“按几次↓”不稳：搜索结果顺序可能变化。
- 高亮状态肉眼/模型容易误读：必须依赖截图复核。

---

## 2) 进入“更全文章列表”（公众号主页/历史消息）
公众号弹窗常只展示“最近推送几条”，不完整。要看更全列表，需要进入公众号主页。

### 步骤
- 在公众号独立窗口中，点击右上角 **人形轮廓/头像** 图标进入公众号主页（历史文章列表）

### 方法：模板匹配（适合固定 UI 图标）
- 对右上角“人形轮廓”这类**稳定 UI 元素**，用模板图（PNG）+ `cv2.matchTemplate` 定位。
- 生成“红框 + 绿点中心”校验图后，再执行点击。

### 坑点：坐标系不一致（非常关键）
- 从截图（如 2940×1912）计算出的坐标，直接用 `cliclick` 点可能不生效。
- 实测需做 **坐标 / 2** 才能命中（Retina/逻辑坐标差异）。
- 建议：把“/2 校准”封装为一次性校验步骤（点一下可验证的按钮并截图确认）。

---

## 3) 在文章列表中定位并进入目标文章（OCR）
文章标题属于动态文本，不适合模板匹配（省略号、热度、缩略图变化、位置变化会导致误匹配）。

### 步骤
- 截取当前文章列表页
- 对截图做 OCR，找到包含关键词的文本块（例如：`将注意力旋转`）
- 输出该文本的 bbox 与 center，并在图上画框
- 人工确认无误后，用 center 点击进入文章（**点击坐标仍需 /2**）

### 方法：EasyOCR
- EasyOCR 返回文本框 4 点坐标 → 计算最小外接矩形 bbox（x_min/x_max/y_min/y_max）→ 取中心点。
- 输出标注图便于复核。

### 坑点
- EasyOCR 首次运行会下载 detection/recognition 模型，等待时间较长。
- 置信度可能不高，但 bbox 往往仍可用；必须用“画框图”复核。

---

## 4) 阅读文章正文直到末尾并总结
公众号文章页通常是内嵌 WebView。

### 步骤
- 进入文章页后，**先点击正文文本区域**让焦点进入 WebView
- 使用 `↓`（key code 125）连续小步滚动，分段截图阅读
- 直到出现 “THE END / 留言区” 等标志确认到达文末
- 汇总要点，输出结构化总结（问题→方法→工程→结果→意义/局限）

### 坑点
- 不先点正文，PageDown/Space 可能不生效（焦点没在 WebView）。
- `Space` 可能误触发“打开图片/GIF”弹层：需按 `Esc` 退出弹层，再点纯文本继续。
- `PageDown`（key code 121）在文章正文页不稳定，最终 **↓ 更稳**。

---

## 5) 最稳可复用套路（强烈建议）
- 固定 UI（按钮/头像）：**模板匹配 + 校验图（红框+绿点）+ 点击（/2）**
- 动态文本（标题/关键字）：**OCR → bbox/center → 校验图（红框+绿点）→ 点击（/2）**
- 读正文：**点击正文聚焦 → Esc 防弹层 → ↓ 连续滚动 + 截图 → 到 THE END**

## 7) UI 操作铁律（必须遵守）
- **任何 UI 操作前**：先截屏确认当前界面状态。
- **任何 UI 操作后**：立刻截屏确认结果是否符合预期。
- **以上两张图都要回传到聊天里**，便于人工复核与回溯。
- **任何需要用 `cliclick` 的点击/拖拽**：必须先基于截图生成标注图（候选 bbox + 中心点），确认无误后再执行，避免误点。

---

## 6) 后续工程化建议
- 封装坐标换算：一次校准确定 `click_scale = 0.5` 或 `1.0`
- 阅读循环：滚动→截图→检测是否到文末（OCR 检测 THE END/留言区关键词）
- 输出结构：标题、作者时间、核心结论 3 条、技术要点、实验数字、适用/不适用场景
